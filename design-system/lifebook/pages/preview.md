# Preview Page Overrides

> **PROJECT:** Lifebook
> **Page Type:** Product Preview / Pre-read

> ⚠️ **IMPORTANT:** Rules in this file **override** the `MASTER.md` file in this same project folder.
> Only deviations from the Master are documented here. For all other rules, refer to the Master.

---

## Page-Specific Rules

### Typography Overrides

- Keep preview typography on the app's sans direction. Do not shift the page into reader-serif styling.
- Use shared dark-surface reading tokens for long-form preview content.
- Hook copy: `soft reading text` (`88%` foreground opacity) at baseline reading line-height.
- Key takeaways: `strong reading text` (`95%` foreground opacity) at baseline reading line-height.

### Content Behavior Overrides

- Preview is optimized for readable skimming, not sustained immersion.
- Truncation is acceptable, but core preview content should remain comfortable to scan on desktop and mobile.
